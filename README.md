Laravel 13 API template built to speed up backend development. It ships with Docker, Sanctum authentication, PostgreSQL, Redis, automated tests, minimum coverage, and static analysis.

## Overview

This project provides a lean foundation for Laravel APIs with ready-to-use Bearer Token authentication, so new APIs can start with login, logout, authenticated user, and protected route flows already in place.

| Item | Description |
| --- | --- |
| Runtime | PHP 8.3 + Laravel 13 |
| Infrastructure | Docker Compose, Nginx, PostgreSQL 16, Redis 7.4 |
| Auth | Laravel Sanctum |
| Quality | PHPUnit, Xdebug Coverage, PHPStan |
| CI | GitHub Actions in a single workflow |

## Quick Start

```bash
cp .env.example .env
make setup
```

Local API:

```text
http://localhost:8080
```

Rebuild the image after changing the Dockerfile or PHP extensions:

```bash
make build
```

## Containers

The `docker-compose.yml` file creates these services:

| Service | Purpose |
| --- | --- |
| `app` | PHP-FPM, Composer, and Artisan |
| `nginx` | HTTP server for the application |
| `postgres` | PostgreSQL database |
| `redis` | Cache, sessions, and queues |

Container names are generated by Docker Compose from `COMPOSE_PROJECT_NAME`. This avoids hardcoded container names and makes it easier to run multiple projects based on this template on the same machine.

Exposed ports:

| Local port | Container | Usage |
| --- | --- | --- |
| `8080` | `nginx:80` | HTTP API |
| `5432` | `postgres:5432` | PostgreSQL |
| `6379` | `redis:6379` | Redis |

## API

Base URL:

```text
http://localhost:8080/api
```

| Method | Route | Auth | Description |
| --- | --- | --- | --- |
| `POST` | `/login` | No | Authenticates and returns a token |
| `POST` | `/logout` | Yes | Revokes the current token |
| `GET` | `/me` | Yes | Returns the authenticated user |
| `GET` | `/protected-test` | Yes | Validates authenticated access |

Login payload after seeding:

```json
{
  "email": "admin@example.com",
  "password": "password"
}
```

## Architecture

The project uses a simple responsibility-based separation:

| Layer | Role |
| --- | --- |
| `Routes` | Declare HTTP endpoints |
| `Controllers` | Coordinate requests and responses |
| `Requests` | Validate input and build DTOs |
| `DTO` | Carry typed data between layers |
| `Services` | Hold business rules |
| `Repositories` | Isolate database queries |
| `Models` | Represent Eloquent entities |
| `Resources` | Format API responses |

Default flow:

```text
Route
  -> Controller
  -> Request
  -> DTO
  -> Service
  -> Repository / Model
  -> Resource
```

Login flow:

```text
POST /api/login
  -> AuthController
  -> LoginRequest
  -> LoginInputDTO
  -> AuthService
  -> UserRepository
  -> LoginResource
```
